## Dependency Chain

```
v0.2 Tool Foundation
 └──► v0.3 Coding Agent (uses ToolExecutor from v0.2)
       └──► v0.3.5 Hardening + Performance (makes v0.3 safe, fast, and consistent)
             └──► v0.4 Codebase Intelligence (plugs deterministic retrieval into ANALYZE)
                   └──► v0.5 Workflow Automation (chains agent runs on top of trusted execution)

Parallel track:
v0.4/v0.5 ──► Model Specialization (optional optimization track)
             • starts during v0.4 with config split, traces, evals, and tool tightening
             • may add a classifier during or after v0.4
             • may add a narrow local-model fine-tune after classifier/eval results
             • does not gate v0.4 or v0.5 delivery
```

## Roadmap Notes

- **v0.3 is feature-complete but not execution-complete.** The next milestone focuses on trust: approvals, rollback correctness, artifact logging, and consistent routing.
- **v0.4 is retrieval-first, not embeddings-first.** File/symbol lookup and deterministic ranking ship before heavier semantic indexing.
- **v0.5 is assisted automation first.** Branch, push, and PR flows require explicit checkpoints until the execution core has proven reliability.
- **Model Specialization is an optimization track, not a product milestone.** It starts during v0.4 with infrastructure work and may improve v0.5 performance, but it must not block shipping either milestone.

## Model Specialization Track

This is a parallel optimization track for small-model routing and local execution on lower-powered devices.

**Starts during v0.4**
- split router model config from executor model config
- enrich run artifacts with session-linked traces, tool trajectories, confidence, and latency
- tighten local tool schemas, prompts, and error messages
- build routing and local-task eval harnesses

**Experiments during/after v0.4**
- benchmark rules vs embedding classifier vs tiny classifier on target hardware
- prototype an optional classifier if baseline routing quality or latency is inadequate

**Possible follow-on after classifier eval**
- fine-tune a narrow local executor for bounded tool workflows only if evals show a clear completion-rate or latency gap

**Exit criteria before classifier training**
- router/executor config split implemented
- session-linked traces captured in artifacts
- routing eval harness exists
- benchmark results on weakest target device exist
- baseline comparison completed for rules vs embedding vs tiny classifier

**Exit criteria before local executor fine-tuning**
- classifier path chosen and benchmarked
- tool-use trajectory dataset collected from real Locode runs
- local-task completion gaps demonstrated by evals
- deployment constraints validated on target hardware
